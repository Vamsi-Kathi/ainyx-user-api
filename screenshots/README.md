# Screenshots

Place the project screenshots in this folder. The main `README.md` references
these exact filenames — keep the names identical (lowercase, `.png`).

| Filename                  | What to capture                                                                 |
|---------------------------|---------------------------------------------------------------------------------|
| `dashboard.png`           | The full `frontend/index.html` console with the Users table populated.          |
| `create-user.png`         | The Create User form filled in (name + DOB) just before/after clicking Create.  |
| `update-user.png`         | A user being edited — form in "Edit User #N" mode with Update button.           |
| `delete-user.png`         | The delete confirmation dialog, or the table right after a delete.              |
| `api-response.png`        | A terminal showing a `curl` call and the JSON response (e.g. GET with `age`).   |
| `docker-running.png`      | `docker compose ps` / `docker compose up` output with both containers healthy.  |
| `tests-passing.png`       | Terminal output of `go test ./... -v` showing all subtests PASS.                |

## Tips for clean screenshots

- Use a maximized browser window and the dark UI for `dashboard.png`.
- For terminal shots, clear the screen first (`clear`) so only the relevant
  command and output are visible.
- PNG format, reasonable resolution (1280px+ wide is ideal for readability on GitHub).
